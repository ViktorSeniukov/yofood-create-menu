# Спецификация: Генератор таблицы корпоративного меню

**Версия:** 1.0  
**Статус:** Draft  

---

## 1. Назначение сервиса

Веб-утилита для офис-менеджеров, которая автоматизирует еженедельную подготовку таблицы корпоративного питания.

**Проблема:** Кейтеринг присылает меню на следующую неделю на сербском языке в виде текстового файла. Офис-менеджер вручную переводит его, структурирует и вставляет в Excel-шаблон, в котором сотрудники выбирают блюда. Это занимает много времени и подвержено ошибкам.

**Решение:** Сервис принимает текстовый файл меню, переводит его через Claude API, структурирует в JSON и автоматически вставляет данные в нужные ячейки Excel-шаблона. На выходе — готовый файл для загрузки в Google Sheets.

---

## 2. Аудитория

Один пользователь — офис-менеджер компании. Авторизация не требуется. Сервис является одностраничной утилитой без хранения пользовательских данных на сервере.

## 4. Экраны и компоненты

### 4.1 Настройки

- Поле ввода Claude API ключа
- Ключ сохраняется в `localStorage` — на сервер не отправляется
- Форма доступна с главного экрана (иконка / ссылка)

### 4.2 Главный экран

Три последовательные зоны:

**Зона A — Загрузка меню**
- Дроп-зона для файла `.txt` или `.docx`
- Файл отправляется в Claude API целиком — фронтенд не парсит и не извлекает текст
- После загрузки — автоматический запрос к Claude API
- Отображение превью переведённой таблицы
- Кнопка «Скачать результат» (опционально)

**Зона B — Загрузка шаблона**
- Дроп-зона для файла `.xlsx`
- Шаблон содержит лист `Tech` и листы по дням недели

**Зона C — Генерация**
- Кнопка «Сгенерировать» — активна только когда загружены оба файла (JSON получен + шаблон загружен)
- После нажатия: фронтенд выполняет мерж и отдаёт готовый `.xlsx` на скачивание

## 5. Claude API — детали интеграции

Claude API задействуется **один раз за весь флоу** — только для обработки меню.

### Задача модели
1. Принять файл меню целиком (содержимое `.txt` или `.docx` — передаётся as-is, без предварительного парсинга на фронтенде)
2. Самостоятельно извлечь из содержимого список блюд по дням и категориям
3. Перевести каждое блюдо на русский язык
4. Сохранить оригинальное название на сербской латинице
5. Структурировать результат в JSON

### Формат ответа (JSON)

```json
{
  "Понедельник": {
    "Завтрак": ["Омлет с беконом и шпинатом / Omlet sa slaninom i spanaćem"],
    "Суп":     ["Телячий суп / Teleća čorba"],
    "Горячее": ["Говяжье рагу с овощами / Juneći ragu sa povrćem"],
    "Гарнир":  ["Картофельное пюре / Krompir pire"],
    "Салат":   ["Салат из свёклы / Cvekla salata"],
    "Десерт":  ["Яблочный пирог / Pita sa jabukama"]
  },
  "Вторник": { ... },
  "Среда":   { ... },
  "Четверг": { ... },
  "Пятница": { ... }
}
```

### Ключи
- Дни недели: `Понедельник`, `Вторник`, `Среда`, `Четверг`, `Пятница`
- Приёмы пищи: `Завтрак`, `Суп`, `Горячее`, `Гарнир`, `Салат`, `Десерт`
- Формат блюда: `Русское название / Naziv na srpskom latiničnom`

## 6. Структура Excel-шаблона (.xlsx)

### Листы
| Лист | Назначение |
|------|-----------|
| `Tech` | Технический лист: имена сотрудников + все блюда по дням и категориям |
| `Пн`, `Вт`, `Ср`, `Чт`, `Пт` | Листы по дням — сотрудники выбирают блюда через селекты |
| `Total`, `By name` | Сводные данные |

### Структура листа `Tech`

Данные сгруппированы по дням недели. Для каждого дня:

| Столбец | Содержимое |
|---------|-----------|
| B | Имя сотрудника |
| C | Фамилия сотрудника |
| D | Формула `=C&" "&B` (полное имя) |
| F | Соки / напитки |
| G | Завтрак |
| H | Суп |
| I | Горячее |
| J | Гарнир |
| K | Салат |
| L | Десерт |

Структура повторяется для каждого дня недели, каждая группа отделена строкой-заголовком с названием дня.

### Мерж (шаг 6)

Операция выполняется **на фронтенде**, без Claude API. Используется библиотека **SheetJS**.

Алгоритм:
1. Взять JSON с переведёнными блюдами из шага 3
2. Открыть загруженный `.xlsx` шаблон через SheetJS
3. Найти в листе `Tech` строки-заголовки для каждого дня
4. Вставить блюда в соответствующие ячейки столбцов F–L
5. Отдать модифицированный `.xlsx` пользователю на скачивание

## 7. Технические требования

### Стек (рекомендуемый)
- **Фронтенд:** Любой фреймворк (React / Vue / Vanilla JS)
- **Интеграция с Claude:** Fetch к `https://api.anthropic.com/v1/messages` напрямую с фронтенда (ключ пользователя)
- **Работа с Excel:** [SheetJS (xlsx)](https://sheetjs.com/) — чтение и запись `.xlsx` в браузере
- **Парсинг файлов меню:** Не требуется — файл `.txt` или `.docx` читается как raw-контент (text / base64) и отправляется в Claude API целиком. Claude сам извлекает и структурирует данные.

### Хранение данных
- API ключ → `localStorage`
- Все файлы обрабатываются в памяти браузера
- Ничего не отправляется на сторонний сервер (кроме запроса к Claude API с содержимым файла меню)

### Модель Claude
- Использовать `claude-sonnet-4-6` (актуальная версия на момент разработки)

## 8. Ограничения и допущения

- Сервис рассчитан на одного пользователя, мультиаккаунтность не нужна
- Формат меню от кейтеринга может меняться — промпт к Claude должен быть устойчив к вариациям форматирования
- Шаблон `.xlsx` офис-менеджер обновляет сам (например, при изменении состава сотрудников)
- Итоговый файл загружается в Google Sheets вручную — автоматической интеграции не требуется
